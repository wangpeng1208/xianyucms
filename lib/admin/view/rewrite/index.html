编号 伪静态格式 支持环境和说明 生成操作

必须开启伪静态并开启自定义伪静态规则并设置好自定义伪静态规则,并且系统自动生成了规则后才能使用：功能状态：{eq name="rewrite" value="1"}可用{else}不可用{/eq}
1. .htaccess格式伪静态 apache或者IIS6.0+Rewrite3的格式 {eq name="rewrite" value="1"}点击生成{else}不可用{/eq}
2. .conf格式伪静态 一般情况为nginx的伪静态格式 {eq name="rewrite" value="1"}点击生成{else}不可用{/eq}
3. httpd.ini格式伪静态 一般情况为IIS6.0+Rewrite2的伪静态格式 {eq name="rewrite" value="1"}点击生成{else}不可用{/eq}
4. web.config格式伪静态 一般情况为IIS7以上支持 请使用.htaccess格式然后导入

1.请尽量将伪静态规则中自定义规则开头为变量参数的规则放到后面，将开头固定字符的规则放在开头,因为伪静态规则是冲上往下匹配所以如果开头为变量的规则容易引起下面其他规则的匹配。


1.生成的规则不能保证100%都能匹配成功，具体容易出现错误就是上面一条所说的规则出现相同的情况了匹配到一个页面中去了，或者出现404

如果有规则不生效，你可以自己修改生成规则中的规则顺序或者修改规则。


2.生成的所有伪静态文保存在runtime/rewrite/中请更具你的环境选择相应规则到你环境中的伪静态文件中,该程序不能保证生成的规则完全能正常使用但是能生成绝大部分的规则
